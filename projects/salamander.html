Salamander

A Vulkan-based 3D rendering engine built with C++20

View on GitHub

Project Overview

Salamander is a 3D rendering engine built with Vulkan using C++20. Developed as a university project for the Graphics Programming 2 course at Howest University, it demonstrates the capabilities of the Vulkan framework while adhering to modern design practices.

Modern Rendering

Deferred rendering pipeline with HDR and tone mapping for realistic lighting.

Advanced Lighting

Directional and omni lights with shadow mapping and Image-Based Lighting (IBL).

Optimization

Double buffering and depth prepass for improved rendering performance.

Rendering Pipeline

Depth Prepass
Albedo Map
Normal Map
Roughness Map
Lighting Pass (HDR)
Post-processing

The Salamander Journey

Developing Salamander was a challenging but rewarding experience during my Graphics Programming 2 course at Howest. Building a Vulkan rendering engine from scratch pushed my understanding of modern graphics APIs and rendering techniques to new levels.

Technical Challenges Overcome

  • Vulkan's explicit memory management required careful handling of resources and layouts
  • Debugging complex synchronization issues between rendering passes
  • Implementing physically-based rendering with accurate material properties
  • Optimizing shadow mapping techniques for directional lights
  • Fixing bugs related to memory layouts, normals, and sampling methods

Despite these challenges, Salamander became a robust platform demonstrating modern rendering techniques, and a foundation I plan to expand into a full 3D game engine.

Technical Highlights

Modern Vulkan Features

  • Synchronization 2 for efficient resource management
  • Dynamic rendering for flexible pipeline creation
  • Vertex pulling using GPU addresses for efficient geometry processing
  • Deferred rendering architecture for complex lighting scenarios
  • Alpha cutout support for complex materials
